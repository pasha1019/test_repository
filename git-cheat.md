## Установка и настройка

### Установка Git
```bash
# Скачать установщик для Windows
https://git-scm.com/download/win

# Проверка версии после установки
git --version
# Пример: git version 2.39.2
```

### Первичная настройка
```bash

# Установка имени (латиницей, в кавычках)
git config --global user.name "User Namovich"

# Установка email (указать свой настоящий email)
git config --global user.email username@yandex.ru

# Просмотр всех настроек
git config --list

# Просмотр файла конфигурации
cat ~/.gitconfig
# Пример:
# [user]
#     name = User Namovich
#     email = username@yandex.ru
```

## Создание и удаление репозитория
```bash
# Создать новый репозиторий в текущей папке
git init

# Удалить репозиторий (стереть папку .git)
rm -rf .git
# Внимание: команда необратимо удаляет всю историю Git
```

## Базовая работа с коммитами
```bash
git status
# Показывает, какие файлы изменены, добавлены в индекс или не отслеживаются
git status --ignored
# Показывает также игнорируемые файлы (из .gitignore)

git add <file>        # Добавить конкретный файл
git add --all         # Добавить все изменения (новые, измененные, удаленные)
git add .             # Добавить все изменения в текущей директории и поддиректориях

git commit -m 'Мой первый коммит!'
# Фиксирует изменения из индекса с указанным сообщением
```

## Просмотр информации и истории
```bash
git log
# Подробная история коммитов

git log --oneline
# Сокращенная история (одна строка на коммит: хеш и сообщение)
# Пример:
# e007f50 Добавить амбиций в список дел
# 8340eb2 Добавить главную страницу

git remote
# Показывает имена связанных удаленных репозиториев
```

## Внутреннее устройство Git
```bash
# Посмотреть текущую директорию
pwd
# /Users/user/dev/first-project
# Зайти в папку .git
cd .git/
# Посмотреть содержимое
ls
# COMMIT_EDITMSG  ORIG_HEAD  description  index  logs/     refs/
# HEAD            config     hooks/       info/  objects/

cat HEAD
# ref: refs/heads/master
# (указывает, что мы находимся на ветке master)

cat refs/heads/master
# e007f5035f113f9abca78fe2149c593959da5eb7
# (хеш последнего коммита в ветке master)
```

## Изменение последнего коммита
```bash
# Добавили файл в индекс
git add common.css
# Дополняем последний коммит (сообщение не меняем)
git commit --amend --no-edit
# Благодаря опции --no-edit сообщение коммита остается прежним

# Проверяем историю
git log --oneline
8340eb2 Добавить главную страницу
# Коммит в истории всё ещё один, но у него новый хеш!
# Меняем сообщение коммита
git commit --amend -m "Добавить главную страницу и стили"
# Проверяем
git log --oneline
a31fa24 Добавить главную страницу и стили
# Сообщение изменилось, появился новый хеш
```

## Отмена изменений
```bash
git restore <file>
# «Откатит» изменения в файле до последней сохраненной версии
# (из последнего коммита или из staging)

git restore --staged <file>
# Переведет файл из staged обратно в modified или untracked
# Пример:
git add example.txt          # Случайно добавили файл
git restore --staged example.txt  # Убрали из staging

git reset --hard <commit_hash>
# «Откатит» историю до коммита с указанным хешем
# Более поздние коммиты и все изменения в рабочей директории потеряются!
```

## Сравнение изменений (diff)
```bash
git diff
# Показывает изменения в файлах, которые еще не добавлены в индекс
git diff --staged
# Показывает изменения, уже добавленные в индекс (подготовленные к коммиту)

git diff <commit1> <commit2>
# Сравнивает два коммита
# Пример: git diff a9928ab 11bada1
```

## Игнорирование файлов (.gitignore)
```bash
# Создать файл .gitignore в корне репозитория
touch .gitignore
# В этом файле перечисляются шаблоны игнорируемых файлов

# Игнорировать системные файлы macOS
.DS_Store

# Игнорировать все файлы с определенным расширением
*.jpeg
*.log
*.tmp

# Игнорировать папку целиком
build/
node_modules/

# Для macOS
.DS_Store

# Игнорировать все файлы, которые заканчиваются на .jpeg
*.jpeg

# Игнорировать все файлы "tmp" во всех подпапках папки docs
docs/*/tmp

# Игнорировать file0.txt, file1.txt и file2.txt
# (но не file3.txt, file4.txt и т.д.)
file[0-2].txt

# Игнорировать todo.txt только в корне репозитория
/todo.txt

# (для сравнения) spam.txt будет игнорироваться во всех папках
spam.txt

# Игнорировать папку build
build/

# Игнорировать файлы docs/current/tmp, docs/old/tmp,
# а также docs/old/saved/a/b/c/d/tmp и даже docs/tmp
docs/**/tmp

# Игнорировать только docs/current/tmp и docs/old/tmp
# (docs/old/saved/a/b/c/d/tmp НЕ игнорируется)
docs/*/tmp

# Игнорировать все JPEG-файлы
*.jpeg

# Но только не мем с Doge
!doge.jpeg
```

## Удаленные репозитории
```bash
# Привязать удаленный репозиторий
git remote add origin https://github.com/user/repo.git

# Показать связанные удаленные репозитории
git remote -v

# Отправить изменения в удаленный репозиторий
git push origin main
```

```bash
# :wq  - сохранить и выйти
# i    - режим вставки
# Esc  - выйти из режима вставки

# Для удобства можно создавать сокращения команд:

git config --global alias.s "status -s"
git config --global alias.cm "commit -m"
---
#HEAD — текущий коммит (последний коммит текущей ветки)
#HEAD~1 или HEAD^ — родитель последнего коммита
#HEAD~3 — прапрародитель (третий коммит назад)
```
